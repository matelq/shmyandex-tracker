# 📋 Shmyandex Tracker

Простой канбан-сервис Shmyandex Tracker: доска с колонками, карточки задач с исполнителями
и описанием, перетаскивание (drag-n-drop), статусы и авторизация.

|                 |                                                                                               |
| --------------- | --------------------------------------------------------------------------------------------- |
| **Фронтенд**    | React + Tailwind CSS + drag-n-drop ([@hello-pangea/dnd](https://github.com/hello-pangea/dnd)) |
| **Бэкенд**      | Node.js + [Fastify](https://fastify.dev/)                                                     |
| **База данных** | SQLite (файл, ничего ставить не нужно)                                                        |
| **Авторизация** | Email + пароль, JWT-токены                                                                    |

---

## 🚀 Быстрый запуск (для всех, даже без опыта)

### Шаг 1. Установите Node.js

Скачайте и установите **Node.js версии 20 или новее** с официального сайта:
👉 **https://nodejs.org/** (кнопка «LTS»). Установка — «Далее → Далее → Готово».

Проверьте, что всё встало. Откройте терминал (на Windows — «Командная строка»
или PowerShell, на Mac — «Терминал») и введите:

```bash
node --version
```

Должна появиться строчка вида `v20.x.x` или выше.

### Шаг 2. Откройте папку проекта в терминале

```bash
cd путь/до/папки/shmyandex-tracker
```

### Шаг 3. Установите зависимости (один раз)

```bash
npm install
npm run setup
```

> Это скачает всё необходимое для фронтенда и бэкенда. Займёт пару минут.

### Шаг 4. Запустите приложение

```bash
npm run dev
```

После запуска откройте в браузере:

## 👉 http://localhost:5173/

Готово! Войдите с демо-аккаунтом или зарегистрируйте свой.

### Демо-аккаунт

При первом запуске автоматически создается пользователь с набором карточек:

|            |                      |
| ---------- | -------------------- |
| **Email**  | `demo@tracker.local` |
| **Пароль** | `demo1234`           |

> Чтобы остановить сервер — вернитесь в терминал и нажмите `Ctrl + C`.

## 🖱️ Как пользоваться

1. **Регистрация / вход** — на стартовом экране введите email, имя и пароль.
2. **Создать карточку** — кнопка «+ Добавить карточку» внизу любой колонки.
   Укажите заголовок, описание, исполнителя и статус.
3. **Перетаскивание** — зажмите карточку мышью и перетащите в другую колонку
   или измените порядок внутри колонки. Статус меняется автоматически.
4. **Редактирование / удаление** — кликните по карточке, чтобы открыть окно
   с кнопкой «Удалить» и сохранением изменений.

Колонки (статусы): **К выполнению → В работе → Готово**.

У каждого пользователя своя личная доска.

## 🛠️ Режим разработки (для разработчиков)

Запуск фронтенда и бэкенда раздельно, с горячей перезагрузкой:

```bash
npm run dev
```

- Фронтенд: **http://localhost:5173** (Vite, автообновление при изменении кода)
- Бэкенд: **http://localhost:3000** (запросы `/api/*` проксируются автоматически)

Откройте в браузере **http://localhost:5173**.

### Полезные переменные окружения (бэкенд)

| Переменная   | По умолчанию                         | Назначение                                    |
| ------------ | ------------------------------------ | --------------------------------------------- |
| `PORT`       | `3000`                               | Порт сервера                                  |
| `JWT_SECRET` | `dev-secret-change-me-in-production` | Секрет для подписи токенов (смените в проде!) |

## 📁 Структура репозитория

```
shmyandex-tracker/
├── package.json          # корневые скрипты (setup / dev / start)
├── backend/              # Node.js + Fastify + SQLite
│   └── src/
│       ├── server.js     # запуск сервера, раздача фронта, JWT
│       ├── db.js         # SQLite, схема таблиц
│       └── routes/
│           ├── auth.js   # регистрация, вход, /me
│           └── cards.js  # CRUD карточек
└── frontend/             # React + Tailwind + Vite
    └── src/
        ├── App.jsx
        ├── api.js                  # обёртка над fetch + JWT
        ├── context/AuthContext.jsx # состояние авторизации
        └── components/             # AuthPage, Board, Card, CardModal
```

База данных — файл `backend/data/tracker.db` — создаётся автоматически при первом запуске.

## 🔌 API (кратко)

Все маршруты карточек требуют заголовок `Authorization: Bearer <token>`.

| Метод    | Путь                 | Описание                                           |
| -------- | -------------------- | -------------------------------------------------- |
| `POST`   | `/api/auth/register` | Регистрация `{ email, name, password }`            |
| `POST`   | `/api/auth/login`    | Вход `{ email, password }`                         |
| `GET`    | `/api/auth/me`       | Текущий пользователь                               |
| `GET`    | `/api/cards`         | Список карточек                                    |
| `POST`   | `/api/cards`         | Создать `{ title, description, assignee, status }` |
| `PUT`    | `/api/cards/:id`     | Обновить (в т.ч. `status`, `position`)             |
| `DELETE` | `/api/cards/:id`     | Удалить                                            |
